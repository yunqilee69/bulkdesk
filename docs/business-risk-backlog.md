# 批发管理系统业务风险与后续功能清单

## 说明

本文记录 2026-07-20 业务审查中确认暂不纳入“退货来源追踪与多角色权限”实施范围的问题。内容用于后续排期，不代表当前系统已经支持。

## 高优先级

### 应收账款与多次收款

当前订单使用单一 `paid_amount` 表示收款结果，未完整表达：

- 多次收款和分期收款。
- 未收余额和客户总欠款。
- 账期、逾期天数和应收账龄。
- 折扣、抹零、坏账和减免审批。
- 收款撤销和冲正。

后续应引入不可变收款流水，并从订单金额、退货金额、收款流水和减免流水计算应收余额。

### 退款与财务对账

本次退货只调整订单累计退货金额、应收净额和客户累计消费，不记录真实资金退款。后续需要支持：

- 原路退款和线下退款。
- 退款凭证与退款操作人。
- 退款失败、撤销和冲正。
- 每日收款、退款和应收对账。

### 经营看板统计口径

当前部分看板以所有非取消订单的原始订单金额计算消费和销量，可能包含未出库、未收款订单，并忽略退货。

后续应明确并分别展示：

- 下单金额。
- 出库金额。
- 应收净额。
- 实收金额。
- 退货金额。
- 净销售额。
- 销售成本和毛利。

## 中优先级

### 采购与供应商结算

当前供应商只作为库存附属资料，没有采购业务单据。后续需要：

- 采购订单。
- 采购收货和采购入库。
- 采购退货。
- 供应商应付账款和付款记录。
- 采购价格历史和供应商报价比较。

### 库存批次与效期

当前库存按商品和仓库聚合，无法同时保存多个供应商、生产日期和有效期批次。后续需要：

- 批次号或批次库存表。
- 每批供应商、生产日期、有效期和库位。
- FIFO 或 FEFO 出库策略。
- 临期、过期和批次召回查询。

### 成本核算

当前入库可以直接修改商品成本价，没有完整成本计算。后续需要确定：

- 移动加权平均成本、批次成本或其他成本方法。
- 销售出库时销售成本快照。
- 退货成本恢复规则。
- 商品和订单毛利报表。

### 数据库库存守恒约束

后续应在数据库增加并验证：

- `quantity >= 0`
- `locked >= 0`
- `locked <= quantity`

服务层校验仍需保留，数据库约束作为最后一道保护。

### 默认等级与默认仓库约束

当前默认等级和默认仓库主要依靠服务逻辑维护。后续需要明确：

- 是否必须始终存在一个默认项。
- 是否允许停用默认项。
- 如何在数据库层防止并发产生多个默认项。

### 接口幂等

订单创建、库存操作、确认收款和退货创建需要业务幂等能力，避免客户端超时重试造成重复单据或重复库存变更。

后续可采用请求幂等键、业务单号唯一键和操作结果重放机制。

## 低优先级

### 赠品业务

标准售价和会员价禁止为零后，赠品应通过独立业务表达，包括：

- 赠品原因和审批。
- 关联销售订单。
- 赠品库存扣减。
- 销售金额为零但成本和数量仍参与审计。

### 客户等级自动变更

当前客户等级只人工维护，`min_spent` 仅记录门槛。后续实现前需要确定：

- 自动升级触发时点。
- 退货后是否自动降级。
- 保级周期和统计周期。
- 人工锁定等级和例外客户。
- 等级变更日志和价格生效时点。

### 报表导出与审计归档

后续可补充订单、库存流水、收款、退货和客户对账单导出，以及长期审计数据归档策略。
