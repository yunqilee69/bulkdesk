# 发货弹窗仓库可分配库存展示设计

## 目标

在订单“开始发货”和“调整分仓”弹窗中，为每个商品的仓库选项显示该仓库当前可分配数量，帮助操作人员在确认分仓前选择有足够库存的仓库。

## 库存口径

展示值使用“可分配数量”，而不是仓库账面实际库存：

```text
可分配数量 = inventory.quantity - inventory.locked + 本订单当前在该商品、该仓库的 reserved 数量
```

加回本订单当前预占，是为了让操作人员调整分仓时能够继续使用原仓库中的已预占库存。其他订单的锁定库存不加回。该数值只用于操作提示，提交时后端仍按加锁后的实时库存进行最终校验。

## 后端接口

新增：

```text
GET /api/v1/orders/{order_id}/shipping-options
```

接口仅允许查询 `placed` 或 `shipping` 状态订单，返回订单商品及各启用仓库的可分配数量：

```json
{
  "items": [
    {
      "order_item_id": "...",
      "product_id": "...",
      "warehouses": [
        {
          "warehouse_id": "...",
          "warehouse_name": "主仓",
          "available_quantity": 12
        }
      ]
    }
  ]
}
```

计算逻辑放在订单服务层，避免前端拼接通用库存接口并重复业务计算。

## 前端交互

- 打开“开始发货”或“调整分仓”弹窗时，同时加载订单详情和发货仓库可分配库存。
- 仓库 Select 宽度由 `260px` 调整为 `360px`。
- 选项文案格式：`主仓（可分配 12）`。
- `available_quantity = 0` 的仓库禁用，不允许新选择。
- 当前明细分配数量超过该商品对应仓库可分配数量时，前端校验提示：`商品 A 在主仓最多可分配 12`。
- 加载失败时不打开可提交的发货弹窗，并提示重新加载，避免展示无库存口径的仓库列表。

## 测试范围

仅覆盖订单模块：

- 后端发货选项库存口径、状态限制和接口契约。
- 前端仓库选项文案、禁用零库存仓库、超量校验、Select 宽度。
- 现有订单开始发货、调整分仓和订单页面测试。
