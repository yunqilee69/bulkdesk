# 商品新建表单修复设计

## 目标

修复新建商品请求可能返回 `500` 的问题，并改善商品表单的图片上传、金额输入和字段排版体验。

## 范围

采用方案 A：在商品页面直接使用 Ant Design 上传和金额输入组件，不新增通用组件抽象。

### 后端

- 创建商品前检查条形码是否已存在。
- 将创建过程中的唯一约束冲突映射为明确的业务错误，避免向前端暴露 `500`。
- 保持现有商品、价格变更日志和图片 URL 列表数据契约不变。

### 前端

- 新建和编辑商品表单改为双列布局；商品描述和图片占满一行。
- 商品图片使用现有 `POST /api/v1/upload` 上传接口，上传前限制图片类型和 10MB 大小。
- 最多上传 9 张图片，支持缩略图预览、删除；提交时只向 `image_urls` 传递成功上传的 URL 列表。
- 标准售价、成本价及价格管理抽屉改用金额输入，限制非负值与两位小数，并显示人民币单位。

## 验收标准

1. 使用有效分类、唯一条形码与完整必填字段创建商品时，接口返回成功并生成两条初始价格变更记录。
2. 重复条形码创建失败时，接口返回可识别的客户端错误，不返回 `500`。
3. 商品图片最多可上传 9 张；每张限图片格式且不超过 10MB；删除后不会提交被删除图片的 URL。
4. 所有价格输入支持最多两位小数、拒绝负数，并在界面显示人民币单位。
5. 商品表单的常规短字段以双列显示，描述和图片使用整行。

## 验证

- 后端：新增或调整商品创建相关测试，运行 `uv run pytest` 及 `PYTHONPYCACHEPREFIX=.pycache python3 -m compileall app`。
- 前端：新增表单值转换/约束测试，运行 `npm run tsc` 与 `npm run biome:lint`。

## 非目标

- 不改变数据库表结构或迁移。
- 不改变上传接口、对象存储策略或图片排序语义。
- 不新增独立的通用图片上传或金额输入组件。
