# 商品列表图片与可销售数量设计

## 背景

商品列表已有商品图片 URL 数据，但表格未展示图片；同时列表没有库存视图，无法快速判断商品是否可销售。

## 目标行为

- 商品列表显示 `image_urls` 的第一张图片作为缩略图；未配置图片时显示统一占位。
- 商品列表显示“可销售数量”。
- 可销售数量按商品在全部仓库的库存聚合计算：`SUM(inventory.quantity - inventory.locked)`。
- 没有库存记录的商品可销售数量显示 `0`。

## 数据与接口契约

- 不新增或修改数据库字段。
- 商品列表响应增加 `available_quantity: int`。
- 后端使用按 `product_id` 分组的库存聚合子查询，并左连接到商品查询。
- 聚合结果使用 `COALESCE(..., 0)`，保证无库存商品返回 `0`。
- 商品分页、关键词、分类、条码、状态筛选和既有分类/品牌名称补充逻辑保持不变。

## 前端交互

- 商品列表新增“图片”列，使用 40px 左右的缩略图且禁用预览。
- 图片 URL 无效时使用组件回退图；没有图片 URL 时渲染文字占位，不显示破损图片。
- 商品列表新增“可销售数量”列，直接显示后端返回的 `available_quantity`。

## 验证范围

- 后端商品列表 SQL 按商品聚合 `inventory.quantity - inventory.locked`，且使用左连接和 `COALESCE`。
- 无库存记录的商品响应为 `available_quantity = 0`。
- 商品 schema 返回 `available_quantity`。
- 前端商品记录类型与表格包含图片和可销售数量字段。

## 非目标

- 不新增仓库维度筛选或分仓库存列。
- 不修改库存锁定、出入库、调拨、盘点或订单流程。
- 不改变商品图片上传、编辑与预览逻辑。
