# 商品级库存预警设计

## 背景

当前 `inventory` 表已经保存 `warning_quantity`，库存列表仅展示该值，不能配置；仪表盘也按单个仓库库存记录判断预警。这与实际业务不符：预警阈值应由商品统一维护，同一商品在全部仓库的可用库存总和低于阈值时才触发一条告警。

库存查询列表还需要显示商品图片，便于快速识别商品。

## 目标行为

- 每个商品保存一个非负整数预警阈值，默认 `0`；`0` 表示不参与库存预警。
- 商品的可用库存为该商品所有仓库的 `quantity - locked` 之和。
- 当商品可用库存小于或等于其预警阈值且阈值大于 `0` 时，仪表盘展示一条商品级库存告警。
- 库存查询列表继续以“商品 + 仓库”展示库存记录，但预警数量读取商品级阈值。
- 库存查询列表展示商品第一张图片；没有图片时显示统一占位状态。

## 数据迁移

1. 为 `products` 增加 `warning_quantity integer not null default 0`，并添加非负约束。
2. 删除 `inventory.warning_quantity` 及其相关注释或约束，不保留或回填旧库存级阈值。
3. 增量脚本归档到 `backend/migrations/incremental/2026-07-17_商品级库存预警.sql`，执行前后检查字段变更结果。

## 接口契约

新增管理员写接口：

`PATCH /api/v1/products/{product_id}/warning-quantity`

请求体：

```json
{
  "warning_quantity": 10
}
```

- 仅允许非负整数。
- 商品不存在时返回 404。
- 成功后返回更新后的商品预警阈值。

库存列表响应新增或保留下列商品派生字段：

- `warning_quantity`：商品级预警阈值。
- `product_image_url`：`image_urls` 的第一张图片，未配置图片时为 `null`。

仪表盘库存预警响应改为商品级数据：每个商品最多一条，包含商品信息、商品图片、总库存、总锁定库存、总可用库存、预警阈值和仓库数量。

## 前端交互

- 库存查询的“商品信息”单元格在商品名称前显示 40px 左右的缩略图；空图片使用 Ant Design 的占位组件，不渲染损坏图片链接。
- 库存查询新增“操作”列，提供“设置预警”。
- 点击“设置预警”打开仅含预警数量的弹窗，输入非负整数并提交。
- 操作从库存记录取得 `product_id`，调用商品级预警接口；保存后刷新库存表，因此该商品的所有仓库行显示一致阈值。
- 仪表盘“库存预警”沿用现有卡片入口，但表格改为商品级告警，不重复显示不同仓库的同一商品。

## 验证范围

- 迁移脚本新增商品阈值字段并移除旧库存阈值列。
- 后端拒绝负数阈值，支持更新商品阈值，并对不存在的商品返回正确错误。
- 库存列表返回商品级阈值和商品图片字段。
- 仪表盘按跨仓可用库存聚合，每个商品最多返回一条告警。
- 前端服务层使用正确接口和字段；库存表展示缩略图、空图占位和预警设置操作。

## 非目标

- 不新增批量预警配置。
- 不改变库存出入库、调拨、盘点或订单锁库的数量计算。
- 不新增站内通知、短信或邮件提醒；本次仅维护仪表盘预警展示。
