# 商品会员价批量维护设计

## 目标

将商品价格管理 Modal 中的会员价维护改为可编辑列表。管理员可以同时查看所有会员等级的价格，为未设置会员价的等级新增价格，并在一次保存中批量更新多个等级，同时查看该商品所有价格类型的变动趋势。

## 范围

- 保留标准售价和成本价的现有单项编辑方式。
- 会员价展示所有会员等级，而不是只展示已存在的 `member_prices` 记录。
- 每个等级显示当前会员价和可编辑的新会员价；缺失记录显示“未设置”。
- 单次保存只提交已新增或价格变化的等级。
- 调整原因改为可选，整批变更共用同一个原因。
- 每个新增或变更的等级仍写入一条 `price_change_logs` 记录。
- Modal 内展示价格变更折线图：标准售价、成本价和每个会员等级会员价各为独立系列。

不包含会员等级管理、会员价删除或独立价格历史页面改版。

## API 设计

新增两个管理员商品接口：

1. `GET /api/v1/products/{product_id}/member-prices`
   - 返回所有会员等级及该商品对应的会员价。
   - 每项包含：`level_id`、`level_name`、`price`（未设置时为 `null`）。

2. `PUT /api/v1/products/{product_id}/member-prices`
   - 请求体：
     ```json
     {
       "reason": "可选的统一调整原因",
       "items": [
         { "level_id": "UUID", "price": 88.5 }
       ]
     }
     ```
   - 校验商品和会员等级存在，价格不得小于 0，等级 ID 不允许重复。
   - 在同一数据库事务中对每项执行 upsert：已有价格更新，缺失价格创建。
   - 每项保存一条会员价变更日志；新增时 `old_value` 为 `null`。空调整原因以空字符串写入现有的非空日志字段。
   - 未提交项目不改动；传入相同价格的项目不生成日志。

不涉及表结构变更，沿用 `member_prices` 的商品与等级唯一约束。

## 前端交互

价格管理 Modal 的会员价区域改为表格：

| 列 | 行为 |
| --- | --- |
| 会员等级 | 显示等级名称 |
| 当前价格 | 显示 `¥xx.xx`，无记录时显示“未设置” |
| 新价格 | 两位小数的价格输入框，带 `¥` 前缀与“元”后缀 |
| 状态 | 标记“新增”“已修改”或“未变更” |

- 首次打开抽屉时加载会员价列表，并以当前价格初始化输入框。
- 用户对未设置等级输入价格即可新增；清空未设置行不提交任何变更。
- 已设置价格不能通过清空删除，避免隐式删除能力。
- 底部的“调整原因”输入框为可选；“保存会员价”仅在存在新增或修改行时可用。
- 保存完成后重拉列表并刷新商品表格；失败时保留用户输入，以便修改后重试。

Modal 底部新增价格变动折线图：

- 读取该商品最近 100 条价格变更日志，横轴为变更时间，纵轴为价格（元）。
- 标准售价和成本价分别标记为“标准售价”“成本价”；会员价按会员等级名称标记为“{等级名称}会员价”。
- 后端日志列表补齐会员等级名称，供图表生成稳定的系列名称。
- 标准价、成本价或会员价保存成功后，自动刷新折线图。

## 验证

- 后端单元测试覆盖：完整列表合并、批量新增、批量修改、混合新增修改、重复等级拒绝、相同价格不写日志、空原因可保存与原子失败回滚。
- 前端单测覆盖：变更行识别、未设置价格显示和空原因可提交。
- 浏览器验证覆盖：打开价格管理 Modal、为多个等级新增/修改价格、一次保存后刷新列表、价格日志和多系列折线图。
