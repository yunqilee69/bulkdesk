# 商品管理系统可靠性修复设计

## 目标

修复当前前后端实现中的业务一致性、接口契约、权限、交互和质量门禁问题，并通过自动化测试与浏览器逐页验证证明主要业务流程及页面按钮可用。环境配置、外部服务部署和生产密钥治理不在本次范围内。

## 业务规则

### 定价

- 每个 SKU 有一个默认售价。
- 每个 SKU 可针对任意会员等级设置独立会员价。
- 创建订单时，若存在当前 `SKU + 客户等级` 的会员价，则使用该价格；否则使用 SKU 默认售价。
- 不创建 `0` 元占位会员价。价格为零仅在用户明确输入零时允许。
- 删除客户等级的 `discount` 字段及前端折扣率输入，避免形成第二套定价规则。

### 订单与库存

- 一个订单请求中同一 SKU 只能出现一次。
- 创建订单锁定库存，发货扣减实际库存并释放锁定，发货前取消仅释放锁定，发货后取消恢复实际库存。
- 库存校验、锁定和订单状态转换使用数据库行锁，防止并发超卖和重复流转。
- 取消订单必须填写原因，并同时保存到订单及状态日志。
- 完成订单更新客户累计消费、订单数、最后下单时间，并且只允许客户等级升级，不允许自动降级。

## 后端设计

- 在 service 层集中实现订单明细去重、库存行锁、订单行锁和状态机；API 层只映射业务错误。
- `OrderOut` 返回页面实际需要的客户名称、订单明细、取消原因和 `from_status/to_status` 日志。
- 仓库 schema 与模型统一联系人、联系电话及 `active/disabled` 状态；创建和更新均保存全部字段。
- 员工列表与详情改用管理员鉴权。仪表盘统计接口直接返回概览数量，避免普通用户调用员工列表。
- 客户模型和初始化迁移增加累计消费与订单数字段；同步更新初始化 SQL 和唯一 Alembic revision。
- 价格日志记录操作人；所有价格修改入口均写日志。
- 上传读取限制为最大尺寸加一个字节，同步对象存储调用放入线程池，避免阻塞事件循环。
- 添加统一异常响应，使业务错误保持 `{code, message, data}` 契约。

## 前端设计

- 服务层维护明确的响应和实体类型，不再依赖不存在的字段或宽泛 `any`。
- 库存分页每页最多 100 条，并通过共享函数收集全部页。请求失败时展示错误并阻止订单或盘点提交，绝不把未知库存当成零。
- 订单取消使用带必填原因的弹窗；订单详情展示客户、SKU、价格、状态流转和取消原因。
- 新建 SKU 只在用户填写具体会员价时提交；移除会员等级折扣率控件。
- 修复仓库禁用枚举、联系人字段、会员价编辑上下文，以及所有已发现的无效按钮参数。
- 替换 Biome 报错和 Ant Design 6 已废弃用法。只在并发锁、库存分页聚合、状态守恒等非直观逻辑处补充简洁注释。

## 错误处理

- 后端将可预期业务冲突返回 400/404/409，并提供统一业务消息；认证和权限错误保留 401/403。
- 前端提交按钮在请求期间禁用，并在 `finally` 中恢复，避免异常后按钮永久锁死。
- 批量操作任一项失败时事务整体回滚，不产生部分库存或价格变更。

## 测试与验收

- 后端测试覆盖定价回退、会员价、重复 SKU、并发锁查询、创建/发货/取消库存守恒、取消原因、客户统计和只升级规则。
- 前端使用 Vitest 与 Testing Library 覆盖各页面的新增、编辑、启停、删除、提交、取消、筛选、分页、上传及错误状态按钮。
- 运行后端编译和 pytest，运行前端 Vitest、TypeScript、Biome、Ant Design lint 和 build。
- 启动前后端后按路由逐页检查管理员与普通用户视角，执行核心链路：基础数据、商品/SKU、会员价、入库、订单、发货、付款、完成和取消。

## 非目标

- 不修改生产数据库地址、Redis/MinIO 地址、默认密钥、CORS 或部署拓扑。
- 不进行与上述可靠性修复无关的页面重设计或目录重构。
